# ZimmerPro — דוח סטטוס, באגים ותוכנית תיקון

**תאריך:** 22 ביוני 2026  
**מטרה:** סיכום למנהל — מה המצב היום, מה חוסם הרצה ופעולות, ומה התוכנית לתיקון.

---

## 1. סיכום מנהלים (30 שניות)

ZimmerPro היא מערכת ניהול יחידות אירוח (React + Express + MongoDB).  
**הבקאנד עולה ומתחבר ל-MongoDB.** **הפרונטאנד עולה** (אחרי תיקון תלות `react-is`).  
**החסימה העיקרית כרגע:** התחברות והרשמה דורשות שירות Inforu (SMS/אימייל) שלא מוגדר — בלי זה אי אפשר לעבור את מסך ה-OTP.  
בנוסף: חלק מהפיצ'רים בממשק לא מחוברים ל-API, יש חוב טכני ממיגרציית מודל נתונים, ואין בדיקות אוטומטיות.

## 3. מפתחות ופרטי סביבה חסרים

קבצים שצריך ליצור/למלא מקומית (לא ב-Git):

### Backend — `backend/.env`

| משתנה | חובה? | סטטוס נוכחי | נדרש ל... |
|--------|--------|-------------|-----------|
| `MONGODB_URI` | **כן** | ✅ מוגדר | כל פעולת DB |
| `FRONTEND_URL` | **כן** | ✅ `http://localhost:5173` | CORS |
| `JWT_SECRET` | מומלץ | ⚠️ placeholder | אבטחת טוקנים |
| `INFORU_USERNAME` | **כן להתחברות OTP** | ❌ placeholder | SMS + אימייל OTP |
| `INFORU_API_TOKEN` | **כן להתחברות OTP** | ❌ placeholder | SMS + אימייל OTP |
| `EMAIL_FROM_ADDRESS` | מומלץ | ❌ placeholder | שליחת OTP באימייל |
| `GOOGLE_CLIENT_ID` | לאימות Google | ❌ placeholder | Google Sign-In |
| `GOOGLE_CLIENT_SECRET` | לאימות Google | ❌ placeholder | Google OAuth בשרת |
| `GEMINI_API_KEY` | לבוט AI | ❌ placeholder | בוט / Gemini |

### Frontend — `.env.local` (שורש הפרויקט)

| משתנה | חובה? | סטטוס | נדרש ל... |
|--------|--------|--------|-----------|
| `VITE_API_URL` | לא | ברירת מחדל עובדת | כתובת API |
| `VITE_GOOGLE_CLIENT_ID` | ל-Google | ❌ חסר | כפתור Google במסך התחברות |
| `VITE_GEMINI_API_KEY` | ל-AI בדפדפן | ❌ חסר | פיצ'רי AI (אם רלוונטי) |

### מה לבקש מהמנהל / ממפתח קודם

1. **Inforu** — `INFORU_USERNAME`, `INFORU_API_TOKEN`, `EMAIL_FROM_ADDRESS` (חוסם התחברות כרגע)
2. **Google OAuth** — Client ID + Secret (אם רוצים התחברות Google)
3. **Gemini API Key** — אם רוצים בוט AI
4. **JWT_SECRET** אמיתי לפרודקשן
5. **משתמש Admin קיים** ב-DB — או אישור ליצור אחד ידנית
6. **אישור לסובב סודות** — נמצאו ערכים רגישים בקבצים ב-repo (`backend/Untitled`, `DEPLOYMENT.md`)

## 8. מבנה הפרויקט (לעיון מהיר)

```
├── App.tsx              → shell, תפריט, auth
│
backend/                 → Express + Mongoose
├── 1-server-express.js  → כניסת שרת
├── 2-routers/           → נתיבי API
├── 3-controllers/       → handlers
├── 4-services/          → לוגיקה עסקית
├── 5-repositories/      → גישה ל-DB
└── models/              → סכמות MongoDB
```
